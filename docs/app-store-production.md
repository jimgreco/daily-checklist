# Production App Store Release

Use GitHub Actions > Publish > Run workflow to kick off a production build.

The Publish workflow always:

- tests, audits, and container-builds the server;
- deploys the server to production and runs the production health check;
- builds the iOS app and uploads the build to TestFlight.

For a release-candidate build, set these manual inputs:

- `update_app_store_metadata`: `true` when `fastlane/metadata` should replace the editable App Store Connect listing fields.
- `update_app_store_screenshots`: `true` when screenshots should be regenerated from deterministic UI-test data and uploaded to App Store Connect.
- `app_store_version`: leave blank unless App Store Connect has an editable version that differs from Xcode `MARKETING_VERSION`.
- `screenshot_devices`: keep the default unless Apple requires another iPhone screenshot size.

## Automated Fields

Fastlane uploads these fields from `fastlane/metadata`:

- name
- subtitle
- promotional text
- description
- keywords
- support URL
- marketing URL
- privacy URL
- release notes
- copyright

Fastlane uploads screenshots generated by the `DailyScreenshots` UI-test target. Screenshots use debug-only seeded local data and do not require a production account.

## Manual App Store Connect Setup

Apple still requires some one-time or review-specific fields in App Store Connect:

- create the app record for bundle ID `com.jimgreco.dailychecklist`;
- set age rating;
- set category and content rights;
- set pricing and availability;
- complete App Privacy answers using `docs/app-store-privacy.md`;
- fill review contact information;
- fill demo account notes if Apple requests credentials;
- choose the uploaded build and submit for review.

Daily sets `ITSAppUsesNonExemptEncryption` to `false`, so the export-compliance answer should match standard platform encryption only.

## Local Screenshot Review

Use Ruby 3.3 or newer, matching the GitHub Actions workflow.

```sh
bundle install
bundle exec fastlane ios screenshots
open fastlane/screenshots
```

To upload reviewed screenshots manually:

```sh
APP_STORE_CONNECT_KEY_ID=... \
APP_STORE_CONNECT_ISSUER_ID=... \
APP_STORE_CONNECT_API_KEY="$(cat AuthKey_XXXXXX.p8)" \
bundle exec fastlane ios screenshots_and_upload
```
